# Sense Article Rich Editor Demo Script

## Demo 路径

### 1. 打开旧文章

步骤：

1. 进入某个已有百科词条
2. 打开 legacy 内容对应修订
3. 进入编辑页

讲解要点：

- 系统会自动识别 `legacy_markup`
- 能保守转换的内容会进入 rich_html 编辑
- 转换失败时不会崩，而是给出只读 fallback

### 2. 展示富文本编辑

步骤：

1. 插入 H2 标题
2. 输入一段正文
3. 给文字加粗、高亮
4. 切换列表与引用块

讲解要点：

- 已从 textarea 升级为现代网页长文编辑器
- 工具栏 sticky，不遮挡正文

### 3. 插入表格与图片

步骤：

1. 插入 3x3 表格
2. 切换为斑马纹
3. 上传图片并设置 alt / caption / 宽度

讲解要点：

- 表格是结构化节点，不是文本占位
- 媒体通过上传后 URL 引用，不走 base64

### 4. Markdown 导入

步骤：

1. 打开“导入 Markdown”
2. 粘贴一段包含标题、列表、引用的 Markdown
3. 导入到当前文档

讲解要点：

- 支持外部内容进入编辑器
- 导入与粘贴治理共同避免脏样式污染正文

### 5. 自动保存与恢复

步骤：

1. 输入几段内容
2. 等待状态区切换到“已保存”
3. 模拟保存失败或关闭标签页后刷新
4. 演示“恢复本地内容”

讲解要点：

- 自动保存不是请求风暴，而是节流更新 draft
- 本地恢复按 revision 隔离，不会串草稿

### 6. 目录导航

步骤：

1. 在长文中插入多个标题
2. 打开预览侧目录导航
3. 点击目录项跳转

讲解要点：

- 基于 headingIndex/rich_html 结构抽取目录
- 已处理 sticky toolbar offset，不会遮挡标题

### 7. compare 与审阅

步骤：

1. 提交修订
2. 打开 review 或 history compare
3. 展示新增段落、表格变化、媒体变化

讲解要点：

- rich compare 不是纯文本 diff，而是块级摘要
- 更适合百科内容审阅

### 8. 触发 warning

步骤：

1. 插入一张没有 alt 的图片
2. 添加空表格或制造标题层级跳跃
3. 查看编辑页状态区和审阅页提示

讲解要点：

- 系统区分 blocking error 和 warning
- warning 可见但不一定阻塞流程

### 9. 审阅通过并发布

步骤：

1. 在 review 页执行通过
2. 完成发布
3. 回到阅读页查看 rich_html 渲染结果

讲解要点：

- 保留既有 revision / review / publish 工作流
- rich_html 发布后可同步兼容 `NodeSense.content`

### 10. 展示维护闭环

步骤：

1. 运行 `npm run test:sense-articles`
2. 运行 legacy 审计脚本
3. 运行媒体审计脚本

讲解要点：

- 这套系统不仅能编辑，也具备迁移、审计、运维与交接材料
- 适合上线验收和团队交接
