# Battle Refactor Phase-1 Checklist

## 目标

验证 `BattleSceneModal` 的 Phase-1 可维护性重构没有改变玩法与表现，并确认新的 hooks / 输入控制器 / 共享数学协议层工作正常。

## 手工验收步骤

1. 打开 PVE 战斗能正常进入 deploy
   - 从正常入口打开战斗弹窗。
   - 确认 `BattleSceneModal` 能显示 HUD、卡牌、小地图、地面场景。
   - 确认控制台没有 WebGL2 初始化错误。

2. deploy 放置 / 移动 / 旋转正常
   - 选中一支可部署部队，点击战场放置。
   - 拖动已选中部队，确认跟随移动正常。
   - 右键拖动部署相机 yaw，确认场景旋转与原行为一致。
   - 滚轮缩放 deploy 相机，确认缩放范围与原行为一致。
   - 若有阵型调整把手，拖动后确认宽度调整正常。

3. 点击开始战斗正常推进
   - 点击 `开战`。
   - 确认 battle phase 正常切换，单位开始推进。
   - 确认 HUD 计时、卡牌状态、结果面板逻辑正常。

4. 小地图点击能定位 / 切换视角
   - deploy 阶段：拖拽中部队可通过小地图点击完成落点。
   - battle 阶段：点击小地图后主视角中心应切换到对应世界位置。

5. 右键移动命令正常
   - battle 阶段选中一支我方部队。
   - 右键点击地面，确认发出移动命令并开始移动。
   - 路径规划 / 技能确认时右键取消逻辑保持原行为。

6. 关闭 modal 不泄露 RAF / interval
   - 打开战斗后再关闭弹窗。
   - 观察控制台，确认没有持续输出或重复渲染相关日志。
   - 再次打开战斗，确认不会出现双重循环、双重 UI 同步或相机状态残留。

## 额外建议

- 开启 debug panel 时观察 `fps / renderMs / simStepMs` 是否与重构前在同一量级。
- deploy 与 battle 分别点击小地图边界位置，确认 `worldToMinimap / minimapToWorld` 协议没有偏移。
