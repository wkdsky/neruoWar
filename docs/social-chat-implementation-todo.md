# 社交与聊天功能实施 TODO

## 目标范围

本 TODO 面向当前阶段的落地工作，目标是在现有系统中逐步实现以下能力：

- 好友申请
- 好友列表
- 私聊会话
- 群聊基础结构
- 右侧抽屉消息入口
- 后续可演进到大规模架构

当前阶段遵循以下原则：

- 先完成 Phase 1 的后端闭环
- 再接右侧抽屉 UI
- 再接 socket 实时推送
- 设计上保留未来拆分为大规模服务化架构的空间
- 当前阶段聊天持久化采用“独立聊天集合”方案：
  - 默认复用主业务 MongoDB
  - 若配置 `CHAT_MONGODB_URI`，则自动切到独立聊天数据库
  - 无论哪种模式，都不写入 `User` 主文档

## 当前硬约束

- [x] 每个用户最多 `200` 个好友

- [x] 私聊数量不设上限
